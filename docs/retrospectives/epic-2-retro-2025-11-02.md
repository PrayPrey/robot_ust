### 🚧 Challenges and Growth Areas

**Blockers Encountered:**

1. **Unicode Encoding Issue (30 min fix)**
   - **Problem:** `°` and `×` characters in PlannerAgent broke 5 tests on Windows
   - **Root cause:** Assumed Linux compatibility, didn't test on Windows early
   - **Resolution:** Changed `°` → `deg`, `×` → `x` in source and tests
   - **Learning:** Always test on target platform from Day 1

2. **Robot Movement Accuracy (2 hour fix)**
   - **Problem:** Pioneer 3-DX only 67% accurate in reaching target positions
   - **Root cause:** Missing wheel kinematic calculations (ω = v/r, differential steering)
   - **Resolution:** Created RobotConfig class with proper formulas
   - **Learning:** Domain-specific physics should be documented in initial stories (Story 1.2)

**Process Gaps:**

3. **Technical Debt Reactive, Not Proactive (John)**
   - Quick Wins weren't in original Epic 2 plan - added reactively when quality dropped to 76.7
   - **Learning:** Schedule explicit tech debt sprints between epics

4. **Evaluation Test Design Complexity (Murat)**
   - Balancing realistic missions vs. test execution speed was challenging
   - Settled on 5 representative missions, but wish we had more edge case scenarios
   - **Learning:** Evaluation test design deserves dedicated story with more time

5. **Underdocumented Libraries (Murat)**
   - pytest-html customization took hours to figure out (custom metadata in HTML report)
   - **Resolution:** Documented approach in code comments for future reference

---

### 💡 Key Insights

**Pattern to Repeat:**
- **Story Context XML is mandatory** - Never start coding without comprehensive context
- **Test-first development** - Write tests before implementation for all acceptance criteria
- **Code review gates** - All stories must be APPROVED before marking done
- **Structured logging early** - Implement logging infrastructure in foundation epic

**Pattern to Avoid:**
- **Assuming cross-platform compatibility** - Test on all target platforms from Day 1
- **Reactive technical debt** - Schedule proactive cleanup sprints
- **Domain knowledge in developer's head** - Document formulas, constraints, limits in AC

**Skills Gained:**
- **Webots kinematic modeling** - Differential drive formulas, wheel velocity calculations
- **pytest-html customization** - Adding metadata, custom CSS, summary sections
- **CrewAI delegation patterns** - Inter-agent communication for complex workflows
- **Production observability** - Multi-sink logging, OpenLit LLM monitoring
